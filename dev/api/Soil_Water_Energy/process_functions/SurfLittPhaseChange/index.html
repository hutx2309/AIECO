Surface-Litter Phase Change

Source: src/Soil_Water_Energy/process_functions/SurfLittPhaseChange.jl

Process-level function for phase change between surface and litter layers.

API

Main.ScriptDocs.SurfLittPhaseChange.surface_litter_phase_change_process — Method
surface_litter_phase_change_process(...)

Calculate vapor-liquid exchange and freeze-thaw in the surface litter micropore domain.

The nonnegative surface litter storages are

\[W_l = \max(0, W)\]

\[V_l = \max(0, V)\]

\[I_l = \max(0, I)\]

where $W$ is liquid water, $V$ is vapor storage, and $I$ is ice.

The free litter pore volume is

\[P_l = \max(0, P - W_l - I_l)\]

where $P$ is total surface litter micropore volume.

When free pore volume is available, the equilibrium vapor concentration is

\[C_v^* = C_v(T, \psi)\]

implemented by

VP_at_psi(surface_temperature, surface_matric_potential)

The potential vapor-liquid exchange is

\[F_v^* = V_l - C_v^* P_l\]

where positive flux means condensation to liquid water and negative flux means evaporation from liquid water.

The vapor-liquid exchange is limited by available liquid water:

\[F_v = \max(F_v^*, -W_l f_r)\]

The associated latent heat flux is

\[H_v = L_v F_v\]

The litter freezing temperature is calculated from water potential:

\[T_f = \frac{-9.0959 \times 10^4}{\psi - L_f}\]

Freeze-thaw is active only when the temperature and available phase storage allow freezing or thawing.

The potential freeze-thaw latent heat is

\[H_f^* = C_l \frac{ T_f - T_l }{ (1 + a_T T_f)(1 - a_\psi \psi) } f_r\]

The limited freeze-thaw heat and liquid-water-equivalent flux are calculated by

freeze_thaw_limited_fluxes(...)

Sign convention

For vapor-liquid exchange:

  • vapor_liquid_flux > 0: condensation to litter liquid water.
  • vapor_liquid_flux < 0: evaporation from litter liquid water.

For freeze-thaw water flux:

  • freeze_thaw_water_flux > 0: ice melts to liquid water.
  • freeze_thaw_water_flux < 0: liquid water freezes to ice.

Legacy behavior note

Current WF6 zeroes both litter evaporation and litter freeze-thaw fluxes when the freeze-thaw activation condition is false. This is physically surprising, because evaporation-condensation and freeze-thaw could be independent. However, to preserve the current WF6 behavior, this function defaults to

zero_vapor_exchange_when_freeze_thaw_inactive = true

Set this to false only if you intentionally want to decouple litter vapor exchange from freeze-thaw activation.

Returns

A named tuple with litter storages, free pore volume, equilibrium vapor concentration, vapor-liquid flux, latent heat, freezing temperature, freeze-thaw flux, and freeze-thaw heat.
